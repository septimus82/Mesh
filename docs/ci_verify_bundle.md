# CI Verify & Artifact Bundle

How to run Mesh Engine verification in CI and publish deterministic
artifact bundles for downstream inspection.

## Quick reference

| Command | Purpose |
|---------|---------|
| `python -m mesh_cli verify-all --artifacts artifacts --ci-bundle` | Run all verification gates, write artifact bundle |
| `python -m mesh_cli artifacts-validate --artifacts artifacts` | Validate the bundle (index + schemas) |
| `python -m mesh_cli verify-report --artifacts artifacts` | Print human-readable diagnostics |
| `python -m mesh_cli verify-all --artifacts artifacts --report` | Same as above, inline after JSON |
| `python -m mesh_cli verify-all --artifacts artifacts --report-json` | Machine-readable diagnostics to stdout |

## GitHub Actions

Minimal workflow that verifies, validates, and uploads:

```yaml
name: Verify
on: [push, pull_request]

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip

      - name: Install dependencies
        run: pip install -e ".[dev]"

      - name: Compile check
        run: python -m compileall -q engine mesh_cli tooling tests

      - name: Tests (fast tier)
        run: python -m pytest -q -W error

      - name: Verify-all + CI bundle
        run: python -m mesh_cli verify-all --artifacts artifacts --ci-bundle

      - name: Validate artifact bundle
        run: python -m mesh_cli artifacts-validate --artifacts artifacts

      - name: Print diagnostics on failure
        if: failure()
        run: python -m mesh_cli verify-report --artifacts artifacts

      - name: Upload artifacts
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: verify-artifacts
          path: artifacts/
          retention-days: 14
```

### Notes

- `--ci-bundle` implies `--report-json-artifact` and `--artifact-index`,
  writing `artifacts/verify_report.json` and `artifacts/index.json`.
- `artifacts-validate` exits 2 on any schema or file-existence issue;
  this fails the job without masking the underlying verify-all exit code.
- The diagnostic step runs only on failure so logs stay clean on green builds.
- Artifacts are uploaded unconditionally (`if: always()`) so you can
  inspect them even on success.

## GitLab CI

```yaml
verify:
  image: python:3.11
  script:
    - pip install -e ".[dev]"
    - python -m compileall -q engine mesh_cli tooling tests
    - python -m pytest -q -W error
    - python -m mesh_cli verify-all --artifacts artifacts --ci-bundle
    - python -m mesh_cli artifacts-validate --artifacts artifacts
  after_script:
    - python -m mesh_cli verify-report --artifacts artifacts || true
  artifacts:
    when: always
    paths:
      - artifacts/
    expire_in: 14 days
```

`after_script` runs regardless of job status, so the human-readable
report is always printed to the log. The `|| true` prevents the
after-script itself from marking the job as failed.

## Local reproduction

Run the same pipeline locally:

```bash
# 1. Full verification with artifact bundle
python -m mesh_cli verify-all --artifacts artifacts --ci-bundle

# 2. Validate the bundle
python -m mesh_cli artifacts-validate --artifacts artifacts

# 3. Human-readable report (optional)
python -m mesh_cli verify-report --artifacts artifacts

# 4. Or inline with JSON output
python -m mesh_cli verify-all --artifacts artifacts --report

# 5. Machine-readable report to stdout
python -m mesh_cli verify-all --artifacts artifacts --report-json
```

### What's in `artifacts/index.json`?

`index.json` is a manifest generated by `--ci-bundle` (specifically
`--artifact-index`). It records:

- **`written`** — every artifact key and its file path (or `null` if not produced)
- **`schemas`** — `schema_version` integers read from each artifact file
- **`readable`** — whether each file exists and parses as valid JSON
- **`generated_files`** — sorted list of all non-null artifact paths

Inspect it with:

```bash
python -m json.tool artifacts/index.json
```

Or validate programmatically:

```bash
python -m mesh_cli artifacts-validate --artifacts artifacts
```

## Updating the baseline

The repo tracks a committed baseline under
`tooling/metrics/ci_baseline_artifacts/`. CI diffs current artifacts
against it to detect regressions.

Refresh it in one shot after a deliberate change:

```bash
# Run verify-all, validate, then copy baseline files
python -m mesh_cli baseline-update

# Or reuse an existing artifacts dir (skips verify-all)
python -m mesh_cli baseline-update --artifacts artifacts

# Then commit the result
git add tooling/metrics/ci_baseline_artifacts && git commit -m 'Update CI baseline'
```
